# Locus Web Viewer

Browser-based terminal viewer for Locus, powered by Plexus RPC.

## Architecture

This is a standalone SPA (Single Page Application) that connects directly to the Locus RPC server via WebSocket.

- **No HTTP server needed** - just serve the static files
- **Pure RPC communication** - uses generated TypeScript client from synapse-cc
- **Real-time updates** - polls RPC server for layout changes

## Generated Client

The TypeScript client in this directory was auto-generated by `synapse-cc`:

```bash
synapse-cc build
```

This reads `synapse.config.json` and generates:
- Type definitions (`types.ts`)
- RPC transport layer (`transport.ts`, `rpc.ts`)
- Namespace clients (`panes/`, `sessions/`, `tabs/`, etc.)
- Bundled ES module (`locus-client.js`)

## Running

### 1. Start the Locus RPC server

```bash
cargo run --bin plexus-locus
# Listening on ws://127.0.0.1:44480
```

### 2. Serve the HTML

Use any static file server:

```bash
# Using Python
python3 -m http.server 8000

# Using Node
npx http-server -p 8000

# Using Bun
bun x serve
```

### 3. Open in browser

```
http://localhost:8000
```

The viewer will connect to `ws://127.0.0.1:44480` automatically.

## Development

### Regenerate client

If the Locus RPC schema changes:

```bash
synapse-cc build
```

This will regenerate all TypeScript files and rebuild `locus-client.js`.

### Configuration

Edit `synapse.config.json` to change:
- Backend URL
- Output directory
- Transport type (browser vs node)

## Files

- `index.html` - Main SPA entry point
- `locus-client.js` - Bundled RPC client (ES module)
- `synapse.config.json` - Code generation config
- `index.ts`, `transport.ts`, `rpc.ts`, etc. - Generated TypeScript source

## How it works

1. HTML imports `locus-client.js` as an ES module
2. Creates RPC client: `createClient({ url: 'ws://127.0.0.1:44480' })`
3. Calls RPC methods: `client.info.dumpLayout()`, `client.panes.capture()`
4. Renders streaming responses in real-time

All communication happens over WebSocket RPC - no HTTP endpoints.
